# UX-редизайн: навигация, настройки при подключённом VPN, избранное

Дата: 2026-07-09. Основание: `wiki/ux-review.md`. Одобрено пользователем.

## 1. Навигация

- Вкладки: **VPN / Конфиги / Маршрутизация / Настройки**. `RoutingScreen` становится вкладкой, его `BreadcrumbBar` убирается.
- Логи: иконка в header-strip Home (справа от `sys.state`) + строка «Логи ›» в секции about Настроек. Открываются push'ем, внутри — `BreadcrumbBar`.
- Из Настроек chevron «Маршрутизация трафика» удаляется.

## 2. Home

- Ячейка «Протокол» → «**Сервер**»: имя активного конфига + address:port. Тап → bottom sheet смены сервера: закреплённые сверху, дальше все конфиги (имя, протокол, пинг), активный подсвечен. Выбор при подключённом VPN → авто-reconnect.
- Пустое состояние: без конфигов подпись «нет конфигурации — добавьте», тап по кнопке питания → переход на вкладку Конфиги.
- Kill switch: отдельное состояние «BLOCKED · kill switch активен» (нативный слой шлёт различимый сигнал; сейчас обрыв выглядит как OFFLINE). Кнопка питания в этом состоянии → reconnect; рядом действие «отключить».
- Фикс дубля «· управляется сервером» в `_ManagedWarning`.

## 3. Настройки при подключённом VPN

- Блокировка `locked` по причине «VPN подключён» убирается полностью. Readonly-профиль остаётся заблокированным; тап по заблокированному ряду → snackbar «Профиль только для чтения».
- Механика — **fingerprint**: при connect `VpnNotifier` сохраняет отпечаток настроек, влияющих на соединение (порт/creds/proxy-only/UDP/ICMP/QUIC/MTU/IPv6/TLS fingerprint/observatory/DNS/routing/split/kill switch/notification). Баннер «настройки изменены · применятся после переподключения [ПЕРЕПОДКЛЮЧИТЬ]» на Настройках и Home при `connected && current != applied`. Возврат настройки назад убирает баннер. Тема/шрифт/автоподключение/UA/автообновление подписок в отпечаток не входят.

## 4. Структура Настроек

```
0x10 appearance   — как есть
0x15 profiles     — как есть
0x20 connection   — автоподключение, boot, уведомление, kill switch, HWID,
                    автообновление подписок (+интервал, UA)
0x30 network  ›   — новый push-экран: random port, порт, random creds, creds,
                    proxy-only, UDP, ICMP, QUIC, TLS fingerprint, IPv6, MTU, observatory
     dns      ›   — DnsSettingsScreen вбирает режим DNS + стратегию
0x40 split    ›   — toggle переезжает внутрь SplitTunnelScreen; в списке chevron
                    со сводкой («выкл» / «N прил., ТОЛЬКО/КРОМЕ»)
0x50 about        — + строка «Логи ›»
```

- При доступном обновлении `_UpdateTile` переезжает наверх списка (единственный экземпляр).

## 5. Конфиги: избранное и жесты

- Группа `[pinned]` первой (над `[local]`), видна только при наличии пинов. Пин = `(subscriptionId?, name)` — переживает обновление подписки (конфиг с тем же именем в той же подписке). Имя исчезло → строка затемнена, «недоступен», действие «Открепить». Пин/анпин — в контекстном меню конфига. Порядок пинов = порядок закрепления.
- «⋮» в строках конфигов и подписок → существующие bottom sheets (long-press остаётся).
- Drag за hex-адрес удаляется. В settings-sheet ⊟ — «Режим сортировки»: в строках появляются ручки ≡ (стандартный drag).

## 6. Полировка

- Общий `PressableRow` (подсветка `accentFade` при нажатии): строки настроек, конфигов, sheet-тайлы, иконки-кнопки.
- Цели нажатия ≥ 40dp; toggle — тап по всей строке.
- `Semantics(label:)`: таббар, кнопка питания, иконки-кнопки.
- AddConfigScreen: при открытии проверка буфера → чип «Вставить из буфера: …».
- Тексты: статусы и console-заголовки — EN, пояснения — RU («tap to connect» → «нажмите для подключения»).
